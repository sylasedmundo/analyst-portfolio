## ЧАСТЬ 2: СЕРГЕЙ НИКОЛАЕВИЧ (специалист 1С)

### API и формат обмена данными

**Рекомендуемый вариант:** HTTP-сервисы (REST) + JSON

**Альтернативы:**
- SOAP веб-сервисы (XML) — устаревший, но работает
- Файловый обмен (FTP/папка) — не рекомендует

**Нужны доработки:** Да  
HTTP-сервисы нужно написать под конкретные задачи  
(типовая конфигурация не содержит их из коробки)

---

### Синхронизация данных

**Основная схема: событийная синхронизация**
- Данные обновляются в момент изменения
- Изменилась цена в 1С → сразу на сайт
- Продали товар → сразу обновился остаток
- Клиент оформил заказ → сразу резервирование в 1С

**Резервная схема: раз в час**
- Страховка на случай сбоев (обрыв связи, зависание)
- Выравнивает данные после восстановления

**Кэширование на стороне сайта:**
- Сайт хранит копию данных у себя
- Клиент видит данные из кэша (быстро)
- 1С обновляет кэш только при изменениях

---

### Данные ИЗ 1С - на сайт

**Товары:**
- Артикул, наименование, описание
- Категория / группа
- Характеристики (память, цвет, диагональ и т.д.)
- Штрих-код, единица измерения
- Статус (активен/архив)
- Фотографий в 1С нет (загружать отдельно)

**Цены:**
- Розничная цена
- Акционные цены
- Дата последнего изменения

**Остатки:**
- По каждой точке отдельно:
  - Склад Подольск
  - Сокол, Медведково, Тёплый Стан, 
    Новокосино, Юго-Западная
- Зарезервированное количество
- Свободный остаток

**Клиенты / Программа лояльности:**
- Номер карты, ФИО, телефон, email
- Накопленная сумма покупок
- Текущий процент скидки (3%, 5%, 7%)
- История покупок

**Статусы заказов:**
- Статус из 1С - на сайт (двусторонний)

---

### Данные С сайта - в 1С

**Заказы:**
- Номер заказа, дата/время
- Состав (товары, количество, цена)
- Сумма (с учётом скидки)
- Способ доставки + адрес
- Способ оплаты + статус оплаты
- Контакты клиента (ФИО, телефон, email)

**Резервирование:**
- Моментально при оформлении заказа
- Снятие резерва при отмене

**Новые клиенты:**
- ФИО, телефон, email
- Привязка к карте лояльности (если есть)
- Если карты нет — 1С создаёт новую, 
  возвращает номер на сайт

**Статусы заказов:**
- Двусторонний обмен (сайт - 1С)
- Статусы: Новый, Подтверждён, Собирается, 
  Передан курьеру, Доставлен, Отменён, Возврат

**Возвраты:**
- Номер заказа, товары к возврату
- Причина возврата, способ возврата денег
- 1С создаёт документ возврата

---

### Обработка ошибок и сбоев

| Сценарий | Решение |
|----------|---------|
| Обрыв при создании заказа | Очередь + автоотправка при восстановлении |
| 1С недоступна | Режим ожидания + очередь запросов |
| Товар закончился | Уведомление клиенту + автовозврат денег |
| Частичная ошибка (часть товаров) | Уведомление с вариантами решения |

**Общие принципы:**
- ✅ Логировать все запросы и ответы
- ✅ Уведомлять администратора (Свету)
- ✅ Не терять данные при сбоях
- ✅ Понятные сообщения клиенту (не "Error 500")
