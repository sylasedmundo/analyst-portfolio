# Функциональные требования
## Онлайн-кинотеатр: Модуль подписок

### FR-001: Оформление подписки
| Поле | Значение |
|------|----------|
| ID | FR-001 |
| Название | Оформление подписки |
| Описание | Система должна позволять пользователю оформить платную подписку на сервис |
| Приоритет | High |
| Связанный UC | UC-1 |

**Детализация:**
- FR-001.1: Система должна отображать список доступных тарифных планов
- FR-001.2: Система должна проверять историю подписок пользователя
- FR-001.3: Система должна предлагать пробный период пользователям, не имевшим подписки 12+ месяцев
- FR-001.4: Система должна поддерживать способы оплаты: СБП, банковская карта
- FR-001.5: Система должна поддерживать применение промокодов
- FR-001.6: Система должна отправлять email-подтверждение после успешной оплаты
- FR-001.7: Система должна активировать подписку в течение 30 секунд после успешной оплаты

---

### FR-002: Отмена подписки
| Поле | Значение |
|------|----------|
| ID | FR-002 |
| Название | Отмена подписки |
| Описание | Система должна позволять пользователю отменить активную подписку |
| Приоритет | High |
| Связанный UC | UC-2 |

**Детализация:**
- FR-002.1: Пользователь может отменить подписку в любой момент
- FR-002.2: При отмене подписка остаётся активной до конца оплаченного периода
- FR-002.3: Система должна отправлять подтверждение отмены на email

---

### FR-003: Автопродление подписки
| Поле | Значение |
|------|----------|
| ID | FR-003 |
| Название | Автопродление подписки |
| Описание | Система должна автоматически продлевать подписку при наличии сохранённого способа оплаты |
| Приоритет | Medium |
| Связанный UC | UC-3 |

**Детализация:**
- FR-003.1: Система должна уведомлять пользователя за 3 дня до автопродления
- FR-003.2: Система должна списывать средства в день окончания подписки
- FR-003.3: При неудачной оплате система должна повторить попытку через 24 часа
- FR-003.4: После 3 неудачных попыток подписка переходит в статус "Истекла"

---

### FR-004: Просмотр истории платежей
| Поле | Значение |
|------|----------|
| ID | FR-004 |
| Название | Просмотр истории платежей |
| Описание | Система должна предоставлять пользователю доступ к истории платежей |
| Приоритет | Medium |
| Связанный UC | UC-4 |

**Детализация:**
- FR-004.1: История должна содержать: дату, сумму, статус, способ оплаты
- FR-004.2: Пользователь может фильтровать историю по периоду
- FR-004.3: Пользователь может скачать чек в формате PDF
